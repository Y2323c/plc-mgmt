作成日: 2026-03-28
最終更新: 2026-04-24（救済コーチングの送信先変更・追加コーチングのリマインド追加・自動送信停止）

## 概要

コーチングリマインドの自動送信基盤を構築した。
前日に送信予定を確認・調整できる仕組みと、GitHub Actions による毎日の自動実行を実装している。

---

## 実装した機能

### 1. GitHub Actions による自動実行

**ファイル:** `.github/workflows/coaching_reminder.yml`

| 時間（JST） | 動作 |
|------------|------|
| 毎日 8:00 | `--preview` モード：翌日の送信予定を完了通知ルームへ通知 |
| 毎日 18:00 | 本番送信：当日対象者のコーチグループルームへリマインド送信 |

手動実行は Actions タブ →「Run workflow」から。`preview` チェックボックスで前日通知モードの手動テストも可能。

---

### 2. `scripts/coaching_reminder.py` の拡張

#### 追加した機能
- `--preview` フラグ：翌日のリマインド対象者を一覧にして管理ルームへ通知
- 全体スキップフラグ確認：`reminder_skip_dates` テーブルに今日の日付があれば全件送信をキャンセル
- 人別スキップ確認：`reminder_skip_targets` テーブルに登録された対象者を除外して送信

#### 実行方法
```bash
cd /Users/Yuka/Desktop/Claude/Claude_code/data-analysis/PLC
python3 scripts/coaching_reminder.py           # 当日送信
python3 scripts/coaching_reminder.py --preview # 翌日予定通知
```

### 3. `pages/13_コーチングリマインド確認.py` の刷新

#### 追加した UI

| 機能 | 説明 |
|------|------|
| 自動送信ステータス | 今日の自動送信が「予定あり」か「キャンセル済み」かを表示 |
| 全キャンセルボタン | 今日の自動送信を全件停止（`reminder_skip_dates` に登録） |
| スキップチェックボックス | 行ごとにスキップを設定。「スキップを保存」で DB に反映 |
| 明日の送信予定フィルター | 前日に翌日の対象者だけ絞り込んでスキップ設定できる |
| 手動送信セクション | 送信前に対象者一覧を表示。ボタンは「リマインドを今すぐ送信（N件）」 |
| 送信後の自動スキップ登録 | 手動送信成功後、自動で skip_targets に登録し 18:00 の二重送信を防止 |
| トースト通知 | スキップ保存後に画面右下に保存完了通知を表示 |

### 4. Supabase に追加したテーブル

#### `reminder_skip_dates`
今日の自動送信を全件キャンセルするためのフラグ。

| 列 | 型 | 内容 |
|---|---|---|
| `skip_date` | date (PK) | キャンセルする日付 |
| `created_at` | timestamptz | 登録日時 |

#### `reminder_skip_targets`
特定の人だけスキップするためのテーブル。

| 列 | 型 | 内容 |
|---|---|---|
| `skip_date` | date (PK) | スキップする送信日 |
| `ticket_id` | uuid (PK) | coaching_tickets.id |
| `session_num` | int (PK) | 対象セッション番号 |

## 日常の運用フロー

```
毎日 8:00 → Chatwork 完了通知ルームに「明日の送信予定」が届く
  │
  ├─ 全員 OK → そのまま待つ
  │
  └─ 調整が必要
       ↓ 13_ページ（Streamlit）を開く
       ↓ フィルター「明日の送信予定」で絞り込む
       ↓ 除外したい人の「スキップ」☑ → 「スキップを保存」
       　 ※ 全員止める場合は「全キャンセル」ボタン

毎日 18:00 → スキップを除いた対象者へ自動送信
```

手動実行: `python3 scripts/monthly_report.py`

### 救済コーチングの送信先（2026-04-24 変更）

救済コーチングはデリケートな事情を含むケースがあるため、**コーチのグループルームではなく運用者専用ルームへ送信**するよう変更。

| 種別 | 送信先 |
|------|--------|
| 新規コーチング | コーチのグループルーム（m_status.room_id） |
| 継続コーチング | コーチのグループルーム |
| 救済コーチング | **運用者専用ルーム（`COACHING_COMPLETION_ROOM_ID`）** |

**変更ファイル：** `scripts/coaching_reminder.py`
- `_collect_targets`（前日プレビュー）と `run`（本番送信）の両方で `coaching_type == "救済コーチング"` の場合に `COACHING_COMPLETION_ROOM_ID` へ送信するよう分岐を追加。

**将来、専用ルームを別に分けたい場合の手順：**
1. `utils/constants.py` に `RESCUE_COACHING_ROOM_ID = "xxxxxx"` を追加
2. `scripts/coaching_reminder.py` の2箇所（`_collect_targets` と `run`）で `COACHING_COMPLETION_ROOM_ID` を `RESCUE_COACHING_ROOM_ID` に置き換え
