## 概要

PLC の運営データ（会員・イベント・コーチング）を Supabase に格納し、Streamlit ブラウザアプリで管理・閲覧できる仕組み。

- **管理者アプリ（`app.py`）:** ローカル起動。事務局のみ使用。
- **公開アプリ（`app_public.py`）:** Streamlit Cloud にデプロイ。会員・当日スタッフが使用。

## テーブル構成

### 1. `users_master`
会員マスターデータ。

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID PK | 自動付与（gen_random_uuid()） |
| member_no | INTEGER | 会員番号 |
| name | TEXT | 氏名 |
| cw_account | TEXT | Chatwork アカウントID（数字） |
| chatwork_id | TEXT | Chatwork ハンドル名（文字列） |
| chatwork_account_id | TEXT | Chatwork アカウントID（API自動取得用） |
| joined_at | TEXT | 入会月（YYYY/MM形式） |
| left_at | TEXT | 在籍有効期限（YYYY/MM形式） |
| left_year | TEXT | 退会日付 |
| birthday | TEXT | 誕生日（MM/DD形式） |
| email | TEXT | メールアドレス |
| activity_type | TEXT | 活動区分（プロデューサー/コンテンツホルダー/両方/未定） |
| note | TEXT | 備考 |
| is_roadmap_active | INTEGER | ロードマップ有効フラグ（0/1） |

### 2. `name_mappings`
システム間の名前表記ゆれを管理。`clean_name` が表示名として最優先で使われる。

| カラム | 型 | 説明 |
|--------|-----|------|

| カラム | 型 | 説明 |
|--------|-----|------|
| id | BIGINT（自動連番） PK | |
| user_id | UUID FK → users_master.id | |

| consult_date | DATE | コンサル実施日（コンサルのみ） |
| note | TEXT | 備考（アンケート備考・チェックイン備考） |

**カテゴリ別カラム使用マップ:**

| カラム | WS | チーム | コンサル |
|--------|-----|--------|---------|
| event_id | ✅ | ✅ | NULL |
| status | 3/4（予定）→ 1/2（確定） | 3/4 → 1/2 | 1 |
| checked_in_by | ✅ | ✅ | NULL |
| consult_type | NULL | NULL | ✅ |
| consult_date | NULL | NULL | ✅ |
| note | ✅（アンケート備考） | ✅ | ✅ |

## 機能詳細

### 出欠アンケート配布フロー（当日までの流れ）

1. `09_アンケート配信` で対象イベントを選択
2. **チェックインURL** をコピーして当日担当者（会員スタッフ）に送付
   - URL例: `https://....streamlit.app/checkin?event_id=xxx`
3. 在籍会員に **アンケートURL** を Chatwork一斉送信 または URLリストから手動送信
   - URL例: `https://....streamlit.app/?event_id=xxx&user_id=yyy`
4. 会員がアンケートに回答（備考欄あり）
5. 当日: 担当者が `/checkin` URLを開き、来場者を順次チェックイン
6. 事務局は `06_出席管理`（管理者アプリ）で状況をリアルタイム確認

### チェックイン（06_出席管理）

- **スマホ対応:** イベント選択はメインコンテンツ表示、ボタンは2列で大きく表示
- **名前フィルター:** 名前の一部を入力して絞り込み可能（スマホで数文字入力するだけ）
- **検索時:** 未回答セクションも自動展開（飛び込み参加者を探しやすい）
- **状態管理:** 参加予定→出席確定 / 欠席予定→欠席確定 / 取消で元に戻せる
- **備考表示:** アンケートで入力された備考をチェックイン画面にも表示

### 会員管理（01_会員管理）

- **重複チェック:** 新規追加時に氏名・メール・Chatwork IDの重複を自動検出してアラート
  - 新規追加: アラートが出ると保存ブロック
  - 編集: アラートは出るが保存は可能（自分自身との一致は除外）
- **Chatwork AccountID 自動取得:** ハンドル名を入力して「自動取得」→ 数字IDを取得・編集可能
- **management_status 99（削除予定）:** 完全削除前の一時保留ステータス

### Chatwork 連携

| 機能 | ページ | 説明 |
|------|--------|------|
| 入会メッセージ送信 | 01_会員管理 | 新規追加後に1対1 DMを送信 |
| アンケートURL一斉送信 | 09_アンケート配信 | 在籍全員にDMで個別URLを送信 |

**`utils/chatwork.py` 関数:**

| 関数 | 説明 |
|------|------|
| `find_account_id(chatwork_id)` | ハンドル名から数字アカウントIDを検索 |
| `get_dm_room_id(account_id)` | 1対1 DM ルームIDを取得 |
| `get_all_dm_room_ids()` | 全DMルームを一括取得（一斉送信時に使用、API節約） |
| `send_message(room_id, message)` | 指定ルームにメッセージを送信 |

> ⚠️ DM ルームは事前に Chatwork 上で相手と繋がっている必要がある。
