## 概要

- **管理者アプリ（`app.py`）:** ローカル起動。事務局のみ使用。
- **公開アプリ（`app_public.py`）:** Streamlit Cloud にデプロイ。コーチ・会員・当日スタッフが使用。

## 機能詳細

### コーチング機能

#### コーチング記録入力（11_コーチング入力.py）— 公開 `/coaching`
- コーチがURLを開いて自分の担当メンバーのセッションを記録する
- URLパラメータ `?coach=コーチ名` で初期選択可能
- タブ: セッション記録 / メモ
- セッション保存後 → Chatwork通知セクションが表示
  - メッセージプレビュー確認・追記が可能
  - 「保存」で記録確定、「📨 Chatworkに送信」で対象グループへ通知
  - 通知は `CHATWORK_COACHING_API_TOKEN`（専用アカウント）を使用
- 各操作後にトーストで完了メッセージ表示

#### コーチング進捗確認（12_コーチング進捗.py）— 公開 `/coaching_status` ＋ 管理アプリ
- タブ1「📈 コーチ別サマリー」: 担当人数・今月の実施・月別ピボット（コーチ選択→他タブに反映）
- タブ2「📊 進捗」: 月フィルター（デフォルト今月）・残り回数・経過日数・次回予定日
- タブ3「💬 メモ」: メモ一覧
- タブ4「📋 実施履歴」: メンバー選択→セッション・メモの全履歴

#### コーチング記録（04_コーチング記録.py）— 管理アプリ
- 有効チケット（is_active=1）のみ表示
- セッション回数は自動計算（手動入力なし）
- タブ: セッション記録を追加 / メモを追加

#### コーチングチケット（03_コーチングチケット.py）— 管理アプリ
- タブ構造: 新規追加 / 既存チケットを編集
- 有効期限: duration_months > 0 → start_date から自動計算。0 → "2099/12/31"（無期限）

### 出欠アンケート配布フロー（当日までの流れ）

1. `09_アンケート配信` で対象イベントを選択
2. **チェックインURL** をコピーして当日担当者（会員スタッフ）に送付
   - URL例: `https://....streamlit.app/checkin?event_id=xxx`
3. 在籍会員に **アンケートURL** を Chatwork一斉送信 または URLリストから手動送信
   - URL例: `https://....streamlit.app/?event_id=xxx&user_id=yyy`
4. 会員がアンケートに回答（備考欄あり）
5. 当日: 担当者が `/checkin` URLを開き、来場者を順次チェックイン
6. 事務局は `06_出席管理`（管理者アプリ）で状況をリアルタイム確認

### チェックイン（06_出席管理）

- **スマホ対応:** イベント選択はメインコンテンツ表示、ボタンは2列で大きく表示
- **名前フィルター:** 名前の一部を入力して絞り込み可能（スマホで数文字入力するだけ）
- **検索時:** 未回答セクションも自動展開（飛び込み参加者を探しやすい）
- **状態管理:** 参加予定→出席確定 / 欠席予定→欠席確定 / 取消で元に戻せる
- **備考表示:** アンケートで入力された備考をチェックイン画面にも表示

### 会員管理（01_会員管理）

- **重複チェック:** 新規追加時に氏名・メール・Chatwork IDの重複を自動検出してアラート
- **Chatwork AccountID 自動取得:** ハンドル名を入力して「自動取得」→ 数字IDを取得・編集可能
- **management_status 99（削除予定）:** 完全削除前の一時保留ステータス
- **入会日の入力形式:** `YYYY/MM/DD`（例: `2023/07/14`）。フォームの日付ピッカーで選択→保存時に自動変換。
  - 既存データの "YYYY年M月" 形式も読み込み時に自動変換される（日は1日として扱う）
  - "継続" など文字列データは変換不可 → 日付ピッカーが今日の日付にフォールバックするため、**必ず正しい日付に修正してから保存すること**

### Chatwork 連携

## セキュリティ設定

### RLS（行レベルセキュリティ）

全テーブルで RLS が有効。公開アプリ（anon key）から操作できる範囲を DB 側で制限している。

| テーブル | anon（公開アプリ） | service_role（管理アプリ） |
|---------|------------------|--------------------------|
| events | SELECT のみ | 全操作 |
| users_master | SELECT のみ | 全操作 |
| name_mappings | SELECT のみ | 全操作 |
| event_logs | SELECT / INSERT / UPDATE | 全操作 |
| m_status | SELECT のみ | 全操作 |
| coaching_tickets | SELECT のみ | 全操作 |
| coaching_logs | SELECT / INSERT | 全操作 |
| message_logs | アクセス不可 | 全操作 |

### API キーの使い分け

| 環境 | キー | 設定場所 |
|------|------|---------|
| ローカル管理アプリ | service_role key | `.env` |
| Streamlit Cloud 公開アプリ | **anon key** | Streamlit Cloud Secrets |

> ⚠️ Streamlit Cloud の `SUPABASE_KEY` は anon key を設定すること（service_role key ではない）。
